## 4. 实现要点

### 4.1 视频上传

抖音网页上传使用隐藏的 `<input type="file" accept="video/*">`。
Playwright 的 `set_input_files()` 可直接绕过隐藏状态：

```python
file_input = page.locator("input[type=file]").first
file_input.set_input_files(video_path)
```

### 4.2 上传进度检测

抖音上传过程中页面会有：
- 上传进度条（`.upload-progress`、`[class*="progress"]`）
- 视频预览区域出现
- loading 动画消失

建议策略：轮询检查"是否有还在显示的 loading/progress 元素"，配合 `video` 标签预览是否出现。

超时时间：120 秒。

### 4.3 标题/描述/话题 页面元素

**话题标签输入注意事项**：
```python
# ❌ 错误：fill 会清除已有内容
tag_input.fill(f"#{tag}")
tag_input.press("Enter")

# ✅ 正确：使用 type_ 追加输入，或先聚焦再 press("End")
tag_input.click()
tag_input.press("End")
tag_input.type_(f" #{tag}")  # 前面加空格隔开
tag_input.press("Enter")
```

抖音创作者后台的页面结构可能随版本变化，**需要根据实际页面调整选择器**。

通用策略（按优先级）：
1. `placeholder` 属性包含"标题"/"描述"/"话题"的 input/textarea
2. `[class*='title']`、`[class*='desc']` 等 class 模糊匹配
3. `page.get_by_role("textbox")` 按角色查找
4. `page.locator("div[contenteditable='true']")` 可编辑 div

话题标签：输入 `#标签名` 后按 `Enter` 或 `,` 确认。

### 4.4 发布后 URL 提取

**URL 跳转检测注意事项**：
创作者后台上传页 URL 本身包含 `creator`，不要用简单的 `"creator" in url` 判断发布成功。建议用更精确的匹配：

```python
# ❌ 过于宽泛
page.wait_for_url(lambda url: "video" in url or "aweme" in url or "creator" in url)

# ✅ 更精确的匹配
patterns = [r'/video/\d+', r'/status/\d+', r'aweme_id=\d+']
page.wait_for_url(lambda url: any(re.search(p, url) for p in patterns))
```

发布成功后页面通常跳转到：
- `https://www.douyin.com/video/{video_id}`
- `https://www.douyin.com/creator/detail/{video_id}`

用正则从 URL 中提取视频 ID：
```python
patterns = [
    r'/video/(\d+)',
    r'/status/(\d+)',
    r'\?video_id=(\w+)',
]
```

## 5. 错误处理策略

| 场景 | 返回 status | message |
|------|------------|---------|
| 视频文件不存在 | `invalid_request` | 具体路径 |
| 未登录 / 无 storage_state | `login_required` | 提示先登录 |
| 上传超时 | `upload_failed` | 超时提示，超时后不得继续流程 |
| 超时后仍返回成功（视频可能已上传） | `upload_failed` | 实际上传可能未完成，需返回失败让调用方重试 |
| 发布按钮未找到 | `element_not_found` | 含选择器信息 |
| 发布被拒（平台报错） | `publish_rejected` | 平台返回的报错内容 |
| 未知异常 | `error` | 异常信息 |
