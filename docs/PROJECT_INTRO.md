# AI Douyin — 短视频内容生成与抖音运营自动化助手

> 一款基于 AI 大模型的抖音内容创作与自动化运营平台，帮助个人创作者和运营团队高效管理短视频内容。
>
> 当前能力细节见 [当前能力总览](CURRENT_CAPABILITIES.md)。

---

## 产品定位

传统抖音运营依赖大量人工操作：写脚本、找素材、配音、剪辑、上传、回复评论……每一项都要花费大量时间。

AI Douyin 的目标是把这些重复性工作**交给 AI 自动完成**，让运营者把精力集中在创意策划和内容质量把控上。

---

## 核心功能

### 1. 内容生成与一键发布

只需输入几个关键词，AI 自动完成全流程：

```
关键词输入
    ↓
PDF/TXT 书籍解析，RAG 知识检索，提取人生智慧/金句
    ↓
AI 生成爆款短视频逐字稿（Hook + Body + 行动建议）
    ↓
AI 语音合成配音（Edge-TTS + GPT-SoVITS 多音色）
    ↓
BGM 背景音乐混音
    ↓
MoviePy 动态字幕、图片转短视频（9:16 1080×1920 MP4）
    ↓
Selenium 模拟浏览器登录抖音，上传、标题/标签填写、发布
```

当前默认主线已切到动漫数字人主讲路线，并可通过浏览器自动化上传到抖音。单人口播模板视频保留为历史/兜底模式。

### 2. 知识库辅助创作

内置古典典籍知识库（论语、孟子等），RAG 语义检索，提取人生智慧/金句为内容增添深度。

```
用户输入关键词 → PDF/TXT 解析 → RAG 语义检索 → 匹配相关知识片段 → AI 结合知识生成脚本
```

### 3. 视频生成路线

当前项目同时保留多条视频路线：

| 路线 | 状态 | 说明 |
|---|---|---|
| 动漫数字人主讲 | 当前主线 | Edge-TTS + Sonic 角色层 + ComfyUI 按需背景 + 字幕合成，当前 `video_mode=presenter_anime` |
| 单人口播模板视频 | 历史/兜底可用 | 管理后台可选“单人口播模板（旧格式）” |
| 双角色/FramePack | 管理后台可选 | 已有样片和合成代码，当前 `video_mode=dual_framepack_active` |
| 手动发布已有 mp4 | 稳定可用 | 适合外部工具生成后由本项目发布 |

ComfyUI 不随管理后台启动；只有生成动漫数字人背景时才按需启动，背景/视频生成完成后代码会尝试关闭。

动漫数字人主讲路线的代表性样片：

```text
data/videos/presenter_20260516_225643_comfy_full.mp4
```

### 4. 自动回复评论

AI 生成回复话术，模拟发送评论/私信，支持意图分类与安全过滤。

```
评论进来
    ↓
意图分类 + 关键词规则匹配（RAG 检索找到最相关的规则）
    ↓
安全过滤
    ↓
固定回复 / 大模型生成回复
    ↓
自动发布至评论区或发送私信
```

### 5. 数据看板

实时查看已发布视频的播放、点赞、评论数据，以及自动回复的覆盖情况。

## 技术特点

| 特点 | 说明 |
|------|------|
| **技术栈** | Python / Selenium 自动化 / Edge-TTS / GPT-SoVITS / MoviePy / RAG / LLM API |
| **本地部署** | 所有 AI 模型和数据都在本地，不依赖云服务 |
| **本地优先** | 大模型可通过 Ollama 本地运行；Edge-TTS 等可选能力需要联网 |
| **安全可控** | 数据本地存储，知识库内容不对外传输 |
| **自动化程度高** | 从脚本生成到视频发布，全流程无需手动切换工具 |

## 适用场景

| 场景 | 解决的问题 |
|------|----------|
| 个人创作者 | 没有团队，独自运营多个账号，时间精力有限 |
| 内容批量生产 | 每天需要发布大量视频，人工操作效率低 |
| 知识类账号 | 需要引用古典文献/专业知识，内容准备耗时长 |
| 评论运营 | 视频多、评论多，人工回复不过来 |

## 团队协作（多用户管理）

管理后台支持多角色分级：

| 角色 | 权限范围 |
|------|---------|
| 超级管理员 | 所有功能、系统配置、用户管理 |
| 运营管理员 | 全部运营操作（发布、评论、自动回复） |
| 运营编辑 | 执行层（手动回复、查看数据） |
| 查看者 | 只读看板数据 |

## 数据安全

- **本地存储**：所有数据（视频、评论、知识库）保存在本地，不上传至第三方
- **磁盘加密支持**：可配合 Windows BitLocker 对存储目录整盘加密
- **访问审计**：检索和操作行为有日志记录
- **知识库脱敏**：导入文档时自动过滤敏感信息

## 快速开始

### 环境要求

- Windows 10/11
- Python 3.9+（建议 3.11）
- 至少 8GB 内存（推荐 16GB）
- NVIDIA 显卡（用于本地 AI 模型加速，推荐非必需）

### 启动步骤

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 .env（复制 .env.example 修改）
copy .env.example .env

# 3. 先生成一段音频或数字人视频做本地验证
python main.py quick --keywords "人生哲学导向"
python main.py presenter --keywords "人生哲学导向" --tts-provider edge --max-segments 16

# 4. 首次发布前，先登录抖音创作者后台
python main.py douyin-login

# 5. 启动管理后台
streamlit run src/web/app.py
```

访问 **http://localhost:8501** 即可使用管理后台。

## 项目结构

```
ai_douyin/
├── src/
│   ├── web/               # 管理后台（Streamlit）
│   ├── platform_adapter/   # 抖音平台对接（浏览器自动化）
│   ├── content_factory/   # 内容生成（脚本、配音、视频合成）
│   │   └── presenter/     # 动漫数字人主讲视频分段、背景、字幕和合成
│   ├── rag_engine/        # 知识库检索（RAG）
│   ├── services/          # 业务服务层
│   └── shared/           # 配置、日志、LLM 客户端
├── data/
│   ├── chroma_db/         # 知识库向量数据库
│   ├── books/            # 导入的书籍/文档
│   ├── videos/           # 生成的视频文件
│   ├── presenter/        # 数字人主讲视频中间产物
│   ├── ip_characters/    # IP 角色素材
│   └── browser/          # 抖音登录状态
└── scripts/
    └── sync_and_import_books.py   # 书籍同步导入脚本
```
