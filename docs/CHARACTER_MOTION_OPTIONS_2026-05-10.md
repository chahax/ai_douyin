# 人物轻微动效方案评估

> 日期：2026-05-10
> 背景：当前 `dual_final_v10.mp4` 的画面已经比前几版稳定，但人物仍偏静态。已试过 `dual_final_v9_motion.mp4` 的“整个人物层轻微漂浮”    ，实际观感不理想，暂不作为主方案。

## 目标

在不破坏当前画面稳定性的前提下，让人物有一点生动感。

核心约束：

1. 不再回到“全身 PNG + SadTalker 头像强行贴合”的旧路线。它已经出现过双头、矩形贴片、错位等问题。
2. 不优先追求完整口型。当前阶段更重要的是画面稳定、自然、能发。
3. 动效幅度宁可小，也不要让人物像漂浮或抖动。

## 已试方案：整个人物层微动

实现方式：

- 在 `src/content_factory/video_composer.py` 的最终 `overlay` 阶段，让角色整体做 `sin()` 周期位移。
- 已生成测试文件：`data/videos/dual_final_v9_motion.mp4`
- 参数级别：上下约 4px，左右约 2px。

结论：

- 技术风险低。
- 但观感不够自然，容易像人物贴纸在背景上滑动。
- 已将 `role_motion` 默认改为 `False`，后续不会默认启用。

适合场景：

- 只用于临时测试，不建议作为正式输出。

## 方案一：背景动，人物不动

思路：

保持人物层完全静止，只强化背景的轻微运动，例如星空缓慢推拉、光圈轻微缩放、粒子飘动、背景慢速横移。

实现方式：

- 对背景视频或背景图做 Ken Burns 效果：
  - 缓慢放大 1.00 到 1.04
  - 缓慢平移 10 到 30px
  - 叠加轻微粒子或光晕变化
- 人物仍使用 `na1_nobg.png`、`n3_nobg.png` 静态叠加。

优点：

- 最稳。
- 不会破坏人物抠图。
- 不会产生双头、贴片、人物边缘错位。
- 观感会比当前静态画面更活，但不会显得假。

缺点：

- 人物本身不动。
- 如果背景太花，可能抢注意力。

推荐程度：高。

建议先试：

- 背景慢推拉。
- 中心光圈做轻微呼吸。
- 人物保持静止。

## 方案二：人物局部 2D 分层动效

思路：

把人物拆成几个独立透明层，只动很小的局部：

- 头部：轻微上下 1 到 2px
- 头发末端：轻微摆动
- 衣服袖口/下摆：轻微摆动
- 身体整体不漂移

实现方式：

1. 用图像编辑或脚本把角色 PNG 拆层：
   - 身体层
   - 头发层
   - 头部层
   - 手臂/衣服边缘层
2. 用 FFmpeg 或 Python/OpenCV 逐帧合成。
3. 每个局部使用极小幅度位移或旋转。

优点：

- 比整个人物漂浮自然。
- 不需要 SadTalker。
- 可以保留当前全身角色画面。

缺点：

- 需要手工拆层。
- 每个角色都要单独处理。
- 角色边缘如果拆得不干净，会出现裂缝或重影。

推荐程度：中高。

适合场景：

- 想继续保留现在这种全身人物站位。
- 能接受一次性做角色素材整理。

## 方案三：LivePortrait / 表情驱动头像版

思路：

放弃全身讲话，改成上半身或头像对话。人物有自然的眨眼、头部微动、表情变化。

实现方式：

- 使用 LivePortrait 类工具对角色头像做轻微表情和头部运动。
- 最终画面设计成：
  - 两个头像/半身对话窗口
  - 背景继续使用星空或抽象场景
  - 字幕和音频照旧

优点：

- 人脸动效比 SadTalker 贴全身更自然。
- 比全身头身分离简单。
- 可以有眨眼、轻微点头、表情变化。

缺点：

- 视觉形态会从“全身角色”变成“头像/半身角色”。
- 需要接入新工具链。
- 角色图需要适合头像驱动：正脸、清晰、无遮挡。

推荐程度：中高。

适合场景：

- 更看重人物“真的在动”。
- 可以接受画面风格变化。

## 方案四：ComfyUI / AnimateDiff 轻动画

思路：

把角色图送入图生视频模型，让模型生成 3 到 5 秒的轻微动态片段，再循环使用。

可尝试效果：

- 轻微呼吸
- 眨眼
- 头发微动
- 衣服轻摆
- 轻微镜头运动

优点：

- 出来的动态可能比手写 FFmpeg 位移自然。
- 能让人物本身有真实变化。

缺点：

- 稳定性不确定。
- 可能改变人物脸、衣服、手部细节。
- 循环点可能不自然。
- 两个角色一致性难控。

推荐程度：中。

适合场景：

- 能接受多试几次，挑最好的一版。
- 对人物一致性要求不是极高。

## 方案五：改成头像对话，直接用 SadTalker 输出

思路：

不再把 SadTalker 的 `256x256` 输出贴回全身图，而是直接把 SadTalker 视频做成头像框或半身对话框。

实现方式：

- 左右各一个头像视频区域。
- SadTalker 输出负责口型。
- 背景、字幕、音频保持当前管线。

优点：

- 口型动效最直接。
- 不再有全身图和头像贴合错位问题。
- 技术边界清晰。

缺点：

- 不是当前全身人物画面。
- 当前角色 B 曾经有 SadTalker 检测失败，需要换更标准的正脸素材。
- 头像框设计不好会显得像会议视频。

推荐程度：中。

适合场景：

- 必须要嘴巴动。
- 可以接受视觉样式重做。

## 方案六：真正头身分离逐帧合成

思路：

保留全身角色，同时让头部或脸部动。需要对每一帧做精确贴合。

实现方式：

1. 为每个角色标定脸部位置、头部大小、脖子位置。
2. 从身体图里去掉原头部，生成无头身体层或遮挡层。
3. 从 SadTalker/LivePortrait 输出中逐帧抠出头脸区域。
4. 每一帧按参数缩放、旋转、贴回身体。
5. 再与背景合成。

优点：

- 理论上最接近“全身角色在说话”。

缺点：

- 开发量最大。
- 每个角色都要单独调参。
- 抠像、脸部边缘、脖子衔接都容易出问题。
- 当前项目之前的问题基本都来自这条路线的简化版。

推荐程度：低，除非后面专门投入时间做这一块。

适合场景：

- 明确要做长期角色 IP。
- 愿意为每个角色建立标准素材和绑定参数。

## 推荐路线

### 第一阶段：先做背景动效

优先试方案一。

原因：

- 当前人物画面已经稳定。
- 背景动效可以明显改善静态感。
- 不会破坏人物。
- 开发成本最低。

建议产出：

- `dual_final_v10_bg_motion.mp4`
- 人物静止。
- 背景慢推拉 + 中心光圈轻微呼吸。

### 第二阶段：做人物局部分层

如果背景动效还不够，再试方案二。

建议先只做一个角色，验证效果：

- 男角色头发或肩部轻微动。
- 女角色头发末端轻微动。
- 不动整个人物位置。

### 第三阶段：决定是否改画面形态

如果必须要明显人物动效，再在方案三和方案五之间选：

- 要自然表情：LivePortrait / 头像版。
- 要口型同步：SadTalker / 头像对话版。

## 不建议继续投入的方向

1. 不建议继续调 `dual_final_v9_motion.mp4` 这种整个人物漂浮方案。它技术上简单，但观感不自然。
2. 不建议继续使用 `data/png/composite_layer.py` 的“抽一帧头像贴全身”逻辑。
3. 不建议把 SadTalker `256x256` 头像视频直接拉伸成全身角色。
4. 不建议在没有拆层的情况下强行做头部替换。

## 下一步建议

先做一个“背景动、人物不动”的 v10。

如果 v10 观感够用，就把它作为当前生产方案；如果还不够，再进入人物局部分层，而不是回到 SadTalker 全身贴合。
