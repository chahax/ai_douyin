# 小说推广视频平台支线设计

更新时间：2026-06-15

## 一句话定位

在现有“动漫数字人主讲视频”主线之外，新增一条“小说推广视频自动化”支线：围绕番茄推广平台账号，自动辅助申请推广名称/任务、获取小说前 10 章内容、生成推广短视频，并把视频与推广任务或推广链接绑定。

## 背景

当前项目主线已经基本实现：

```text
关键词/文章
  -> 口播稿
  -> 分段配音
  -> 动漫背景
  -> Sonic 数字人层
  -> 字幕合成
  -> 视频输出/抖音发布
```

新的业务目标不是普通知识口播，而是“小说分销/推广视频”：

```text
番茄推广平台账号
  -> 申请/选择推广任务
  -> 获取小说基础信息和前 10 章内容
  -> 提炼卖点、冲突、钩子和推荐理由
  -> 生成小说推广视频
  -> 绑定推广链接/任务
  -> 输出可发布视频和投放记录
```

因此该能力应作为独立支线建设，不直接侵入 Presenter 主线。

## 中国系统中的业务定位

该支线在中国内容系统中属于“内容分发 + CPS/CPA 推广 + 短视频素材生产”的交叉模块。

建议定位为：

```text
小说推广视频自动化支线
```

与现有系统关系：

| 模块 | 当前定位 | 与支线关系 |
|---|---|---|
| Presenter 动漫数字人主线 | 通用口播视频生产 | 作为推广视频合成能力复用 |
| 抖音发布自动化 | 已有发布能力 | 后续可接推广视频发布 |
| 评论/自动回复 | 运营能力 | 后续可用于推广视频评论承接 |
| RAG/文章提炼 | 文本理解能力 | 可复用为小说章节提炼 |
| 新增小说推广平台支线 | 推广任务和素材生产 | 独立任务、独立数据、独立合规边界 |

## 可行性分析

### 可行部分

以下能力技术上可行：

| 能力 | 可行性 | 说明 |
|---|---|---|
| 推广任务信息录入 | 高 | 手动录入或自动读取任务页字段 |
| 自动申请推广名称 | 中 | 前期优先浏览器自动化；后续可在明确接口和授权边界后演进为 JS 逆向/API 调用 |
| 小说前 10 章内容获取 | 中 | 前期可通过网页搜索、公开视频页面或可访问章节页抓取验证；需保留来源记录 |
| 小说卖点提炼 | 高 | 可用 LLM 从章节生成钩子、冲突、人物关系 |
| 推广脚本生成 | 高 | 可复用现有文章提炼/Presenter 口播链路 |
| 推广视频生成 | 高 | 可复用 Presenter 合成链路，新增小说模板 |
| 绑定视频 ID/任务 | 中 | 番茄推广平台侧需要绑定短视频平台的视频 ID；前期可半自动录入视频 ID 后绑定 |
| 发布到抖音 | 中高 | 已有抖音发布自动化，可扩展标题/话题；前期没有挂载权限时只能在评论区发送关键词/搜索引导，后续有权限后再挂链接 |

### 不确定部分

以下部分必须先验证，不应直接承诺全自动：

| 不确定点 | 风险 | 验证方式 |
|---|---|---|
| 番茄推广平台是否有公开 API | 如果无 API，前期用浏览器自动化；后续可评估 JS 逆向/API 调用 | 抓包、文档、页面调研，记录接口、签名、token、风控触发点 |
| 是否有验证码/滑块/短信验证 | 自动申请可能被阻断 | 手动走一次完整流程；保存用户授权后的登录态/用户信息，减少重复登录 |
| 是否允许批量获取章节内容 | 版权与平台规则风险 | 先通过网页搜索和可访问章节页抓取验证，记录来源 URL、章节范围和抓取时间 |
| 推广平台绑定方式 | 平台侧可能要求绑定短视频视频 ID | 记录推广平台绑定视频 ID 的入口、字段、状态和失败原因 |
| 短视频平台是否允许挂载该类链接 | 发布受平台规则影响 | 前期没有推广挂载权限时，评论区不能挂链接，只能发送关键词/搜索引导；后续有权限后再验证挂链接 |

### 自动化演进路线

番茄推广平台接入建议分三层推进：

| 层级 | 方式 | 适用阶段 |
|---|---|---|
| 第一层 | 浏览器自动化 | 快速验证流程，适合登录、任务申请、视频 ID 绑定 |
| 第二层 | 页面接口调用 | 当接口稳定、参数清晰、账号授权可控时，用 API 化方式替代部分页面点击 |
| 第三层 | JS 逆向辅助 | 当接口存在签名、加密参数或动态 token 时，用逆向结果还原必要请求参数 |

约束：JS 逆向只用于用户自己账号的授权业务自动化，不用于绕过验证码、登录安全、权限控制、付费墙或平台风控。

## 合规边界

该支线必须内置合规约束，否则后续自动化风险很高。

### 必须遵守

- 只处理用户有权推广的小说任务。
- 只使用平台允许展示、下载、复制或推广的章节内容。
- 不绕过验证码、登录安全、访问限制或付费墙；只保存用户主动授权后的登录态和必要用户信息。
- 不伪造平台接口，不篡改推广归因参数。
- 不生成虚假承诺，例如“免费看全集”“官方保送”“必赚收益”。
- 不生成擦边、低俗、夸大、诱导未成年人付费等违规营销内容。
- 发布前保留人工审核开关。

### 建议产品规则

- 默认采用“半自动审核模式”：自动生成素材，但申请任务、绑定视频 ID、评论区关键词/搜索引导和最终发布保留人工确认。
- 所有推广任务保存来源、任务 ID、小说 ID、推广链接和生成时间。
- 所有平台账号会话只保存必要登录态，不保存明文密码；失效后由用户重新登录授权。
- 所有视频脚本保存对应章节范围和生成 prompt，方便追溯。
- 不把小说全文长期存入知识库；只保存必要摘要、章节编号和任务元数据。

## 总体流程设计

### 人工处理入口

支线不是一开始全自动。以下节点需要保留给用户处理或确认，系统只做辅助、记录和生成：

| 入口 | 用户处理内容 | 系统辅助内容 | 阶段 |
|---|---|---|---|
| 小说内容获取 | 用户提供前 10 章文本、章节网页、搜索结果或允许访问的来源 | 清洗章节、去广告、保存来源、提炼剧情 | MVP |
| 推广任务信息 | 用户提供任务页、小说名、任务 ID、推广名称要求 | 结构化任务 JSON、保存任务状态 | MVP |
| 评论关键词生成 | 用户确认评论区可发送的关键词、搜索词、引导语 | 生成多个关键词/搜索引导版本，过滤违规词 | MVP |
| 视频发布确认 | 用户确认标题、简介、话题、视频内容 | 生成发布文案和审核清单 | MVP |
| 视频 ID 获取 | 用户发布后提供短视频平台视频 ID，或授权系统读取 | 记录视频 ID，关联推广任务 | 第二阶段 |
| 推广平台绑定 | 用户确认在推广平台绑定该视频 ID | 浏览器自动化打开页面、填入视频 ID、截图留痕 | 第二阶段 |
| 后续挂链接权限 | 用户确认账号是否已有挂链接/组件权限 | 有权限后切换为链接挂载流程，无权限继续关键词引导 | 第三阶段 |

第一阶段优先把这些入口做成显式输入，不做隐式全自动，避免平台权限、验证码、内容来源不确定导致主流程不稳定。

### MVP 流程

```text
手动登录番茄推广平台
  -> 手动或自动录入推广任务信息
  -> 用户提供或确认小说前 10 章内容来源
  -> LLM 提炼剧情钩子和卖点
  -> 生成 30-60 秒推广口播稿
  -> Presenter 生成推广视频
  -> 生成标题、简介、话题、推广链接记录
  -> 人工审核后发布
  -> 获取短视频平台视频 ID
  -> 到推广平台绑定视频 ID
  -> 用户确认评论区关键词/搜索引导后发送
```

### 目标流程

```text
平台账号会话检查
  -> 自动进入推广平台
  -> 自动申请或选择推广名称
  -> 自动读取可推广小说列表
  -> 自动获取指定小说前 10 章
  -> 自动生成多个推广视频方案
  -> 自动发布或辅助发布到短视频平台
  -> 获取短视频平台视频 ID
  -> 自动绑定推广平台任务和视频 ID
  -> 前期无挂载权限时自动生成评论区关键词/搜索引导文案
  -> 人工审核
  -> 同步视频数据和转化记录
```

## 内容生成策略

小说推广视频不能简单复述章节，需要按短视频转化逻辑组织。

### 推荐脚本结构

```text
3 秒强钩子
  -> 主角身份/困境
  -> 核心冲突
  -> 反转或爽点
  -> 悬念断点
  -> 引导阅读/搜索/点击
```

### 内容风格

| 类型 | 适用小说 | 特点 |
|---|---|---|
| 剧情解说型 | 大多数小说 | 稳定、安全、适合数字人口播 |
| 悬念钩子型 | 悬疑、都市、逆袭 | 前 5 秒强冲突 |
| 人设爽点型 | 玄幻、都市、重生 | 强调主角身份变化和反击 |
| 情绪共鸣型 | 现实、情感 | 强调委屈、误会、选择 |

### 禁止内容

- 不剧透核心大结局。
- 不冒充官方作者或平台客服。
- 不承诺收益、返现、免费全集。
- 不使用低俗擦边描述吸引点击。
- 不编造小说不存在的桥段作为事实。

## 视频生成策略

MVP 阶段建议直接复用 Presenter：

```text
小说前 10 章
  -> 小说推广口播稿
  -> PresenterRequest(input_mode=article_direct 或内部 text)
  -> 分段音频
  -> 背景图
  -> Sonic 数字人层
  -> 字幕合成
```

后续可以新增 `video_mode=novel_promotion`，但第一阶段不建议重做合成器。

建议视频元素：

- 右下角 Sonic 或后续专属推广角色。
- 背景使用小说类型相关场景，例如都市、古风、玄幻、校园、悬疑。
- 字幕强调钩子、冲突和悬念。
- 结尾保留推广引导文案，但不直接做违规诱导。

## 开发阶段

### 第 0 阶段：平台调研和流程录屏

目标：确认番茄推广平台真实流程。

交付物：

- 平台登录方式记录。
- 申请推广名称流程记录。
- 小说任务页字段清单。
- 前 10 章内容来源确认，包括平台页、网页搜索结果、可访问章节页。
- 推广平台绑定视频 ID 的入口、字段和状态确认。
- 风控点记录，例如验证码、短信、滑块、二次确认。
- 登录态保存方式确认，包括 cookie/localStorage/sessionStorage 及失效时间。

验收标准：

- 能手动完成一次从任务申请、视频发布、获取视频 ID、推广平台绑定视频 ID 的完整闭环。
- 能确认哪些步骤适合自动化，哪些必须人工确认。

### 第 1 阶段：离线 MVP

目标：不碰平台自动化，先把“小说内容 -> 推广视频”跑通。

范围：

- 手动创建任务 JSON。
- 手动提供前 10 章文本。
- 自动生成推广脚本。
- 自动生成 Presenter 推广视频。
- 输出标题、简介、话题和推广链接记录。

验收标准：

- 一条命令生成一个可审核推广视频。
- 不依赖浏览器自动登录。
- 所有产物进入 `data/novel_promotion/`。

### 第 2 阶段：半自动平台接入

目标：自动读取任务信息、保存登录态、辅助绑定视频 ID，但关键操作人工确认。

范围：

- 浏览器自动化打开番茄推广平台。
- 复用用户授权后的登录态，失效时提示用户重新登录。
- 读取任务页基础信息。
- 自动填入推广名称草稿。
- 自动打开绑定入口并填入短视频平台的视频 ID。
- 前期无挂载权限时，生成评论区关键词/搜索引导文案；后续有权限后再生成挂链接文案。

验收标准：

- 不绕过验证码和安全校验。
- 用户确认后再提交申请、绑定视频 ID、发布评论区关键词/搜索引导。
- 失败时保留截图和错误日志。

### 第 3 阶段：批量生成和发布

目标：支持多小说、多脚本、多视频批量生产。

范围：

- 批量任务队列。
- 每本小说生成多个钩子版本。
- 自动去重标题和封面风格。
- 接入抖音发布自动化。
- 记录发布结果。

验收标准：

- 每个任务可追溯章节来源、脚本版本、视频路径、短视频视频 ID、发布账号、推广链接和评论区引导状态。
- 支持人工审核队列。

### 第 4 阶段：数据闭环

目标：根据播放、点击、转化数据优化脚本和视频模板。

范围：

- 同步短视频平台播放/评论数据。
- 同步推广平台点击/转化/收益数据，前提是平台允许。
- 形成小说题材、钩子类型、视频风格的效果分析。
- 自动推荐下一批生成策略。

验收标准：

- 能按任务查看投入产出和素材效果。
- 能标记高转化脚本模板。

## 优先级建议

建议先做第 1 阶段离线 MVP，原因：

- 可以最快验证“小说前 10 章生成推广视频”的内容质量。
- 不受番茄推广平台风控、验证码、页面变化影响。
- 能直接复用现有 Presenter 主线能力。
- 后续平台自动化失败时，离线视频生产能力仍然可用。

推荐第一个可开发任务：

```text
新增 novel-promo video 命令：读取本地任务 JSON + 前 10 章文本，生成推广脚本和 Presenter 视频。
```

## 主要风险

| 风险 | 等级 | 缓解 |
|---|---|---|
| 平台无 API 且强风控 | 高 | 先浏览器自动化，保存授权登录态；后续再评估 JS 逆向/API 调用 |
| 小说内容抓取存在版权/协议限制 | 高 | 只处理授权任务和可推广内容 |
| 自动生成文案夸大或违规 | 中高 | 加审核、禁用词和合规 prompt |
| 视频批量同质化 | 中 | 多模板、多钩子、多背景风格 |
| 推广平台绑定视频 ID 失败 | 中 | 本地记录任务、视频 ID 和失败截图，人工兜底 |
| 页面结构频繁变化 | 中 | 浏览器自动化封装在独立模块 |

## 当前不做

- 不做绕过验证码、滑块、短信验证；只复用用户授权后的有效登录态。
- 不做小说全文爬虫系统。
- 不把章节全文导入长期 RAG 知识库。
- 不直接批量自动发布未经审核的视频。
- 不在第一阶段重做视频合成器。
