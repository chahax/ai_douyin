# TTS 语音合成框架对比与迁移方案

## 1. 现状评估
当前项目使用的是 **Edge-TTS** (微软 Azure TTS 的免费接口)。
*   **优点**: 免费、无需 API Key、响应速度快、支持基础情感调节。
*   **缺点**: 音色库固定，无法微调参数（如磁性、共鸣感），容易被平台识别为 AI 配音（同质化严重）。

## 2. 替代方案对比

| 方案 | 特点 | 磁性/成熟度 | 成本 | 部署难度 | 推荐指数 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **GPT-SoVITS (推荐)** | 开源、支持**5秒样本克隆**、音色无限、情感表现力极强 | ⭐⭐⭐⭐⭐ (可定制任意磁性声音) | 免费 (需本地显卡) | 中 (需 Python 环境 + 4G+ 显存) | ⭐⭐⭐⭐⭐ |
| **ChatTTS** | 专注于对话场景，有呼吸声/停顿，自然度高 | ⭐⭐⭐⭐ | 免费 (需本地显卡) | 中 | ⭐⭐⭐⭐ |
| **Fish Audio** | 在线 API，效果惊艳，支持声音克隆 | ⭐⭐⭐⭐⭐ | 付费 (有免费额度) | 低 | ⭐⭐⭐ |
| **OpenAI TTS** | `tts-1-hd` 模型，音质极高，但风格较固定 | ⭐⭐⭐⭐ | 付费 ($0.03/1k chars) | 低 | ⭐⭐⭐ |
| **CosyVoice** | 阿里开源，多语言支持好，自然度高 | ⭐⭐⭐⭐ | 免费 (需本地显卡) | 中 | ⭐⭐⭐ |

## 3. 为什么选择 GPT-SoVITS？
对于“人生激励”类短视频，**声音的磁性和感染力是核心竞争力**。
*   **定制化**: 你可以找一段你喜欢的电台男主播/电影旁白的音频（只需5-10秒），直接克隆出他的音色。
*   **零成本**: 完全本地运行，不用担心 API 费用。
*   **可控性**: 可以精确控制语速、语调，甚至可以通过“参考音频”来控制情感（比如输入一段悲伤的参考音频，生成的语音也会带有悲伤感）。

## 4. 迁移实施计划

### 第一阶段：本地部署 GPT-SoVITS
1.  **环境准备**: 确认本地机器是否有 NVIDIA 显卡 (建议 4G 显存以上)。
2.  **安装**: 拉取 GPT-SoVITS 开源仓库，安装依赖。
3.  **模型准备**: 下载预训练模型。

### 第二阶段：声音克隆 (核心)
1.  **素材获取**: 找一段 5-10 秒的纯净人声（无背景音乐），必须是您理想中的“磁性成熟男声”。
2.  **微调 (可选)**: 如果对效果要求极高，可以录制 1 分钟音频进行快速微调 (1-shot tuning)。
3.  **推理测试**: 在 WebUI 中测试生成效果。

### 第三阶段：代码集成
1.  **API 封装**: GPT-SoVITS 提供了一个 API 服务 (`api.py`)。
2.  **适配器模式**: 修改 `src/content_factory/tts_engine.py`，增加 `GPTSoVITSAdapter`，保持原有接口不变，底层切换为调用本地 API。

### 4.1 目录结构变更
```
src/
└── content_factory/
    ├── tts_engine.py       # 主入口 (工厂模式)
    ├── tts_providers/
    │   ├── edge_tts.py     # 现有实现
    │   └── gpt_sovits.py   # 新增实现
```

## 5. 风险与对策
*   **推理速度**: 本地生成速度可能不如 Edge-TTS 快。 -> **对策**: 异步生成，提前缓存。
*   **硬件限制**: 如果没有显卡，只能用 CPU (非常慢)。 -> **对策**: 使用 Colab 部署或租用 GPU 云服务器。
